Introduction¶

Dataset Description¶

This project combines two public datasets:

1. FBI NICS Gun Background Check data (gun_data.csv) — 12,485 rows × 27 columns, covering monthly background-check counts per U.S. state/territory from November 1998 through September 2017. Each row is one (state, month) pair. Key columns include:

  • month — the reporting month in YYYY-MM format
  • state — U.S. state, territory, or D.C.
  • permit, permit_recheck — concealed-carry permit application and renewal checks
  • handgun, long_gun, other, multiple — checks initiated for each firearm type
  • redemption_*, rentals_*, private_sale_*, return_to_seller_* — checks tied to specific transaction types (many were added in later years and are sparse)
  • totals — total background checks initiated that month in that state (the main dependent variable)

2. U.S. Census data (us_census_data.csv) — a single snapshot of state-level demographic indicators (primarily July 1, 2016 estimates). The file is in wide format: rows are demographic "Facts" (population, age %s, race %s, etc.) and columns are states.

The two datasets relate to one another by state name, allowing us to compute a per-capita view of gun-related activity and to compare check activity against demographic composition.

Questions for Analysis¶

This report investigates two research questions:

  1. RQ1 — How have FBI gun background checks trended over time, and which states drive the most activity? This explores the totals dependent variable against the independent variables time (month), state, and population. We look at national monthly trends, raw state totals, and population-normalized rates.

  2. RQ2 — Do state demographics relate to per-capita gun background check rates? Using the merged gun + census dataframe, we relate the dependent variable checks_per_100k against three demographic independent variables: pct_65plus, pct_under18, and pct_white, then look at a broader correlation matrix.

Because all analyses are descriptive/correlational, findings are presented as associations, not causal claims.

Data Cleaning¶

Having inspected the raw gun data above, the cells below apply three cleaning steps, each documented in markdown next to the code that performs it.

Based on the initial exploration, the gun dataset has three issues that need attention:

  1. month is stored as a string — we convert it to a proper datetime so we can do time-based analysis.
  2. Many columns have missing values — columns like other, permit_recheck, rentals_*, and returned_* are sparse because the FBI added those reporting categories in later years. We fill these with 0 since a missing value here means "not yet reported," not "unknown amount."
  3. Duplicates — we check for and drop any duplicate (state, month) rows.

    frame[columns] = frame[columns].fillna(0)

Loading the U.S. Census Data¶

The census dataset is structured in wide format — each row is a demographic fact and each column is a U.S. state. We need to transpose it so that each row represents one state, then extract the key columns we'll use for analysis (population and demographic percentages).

In [12]:
# Load the census data (wide format: rows=facts, columns=states)
census_raw = pd.read_csv('us_census_data.csv')

# Transpose so rows = states, columns = facts; set state names as index
census_T = census_raw.set_index('Fact').drop(columns=['Fact Note']).T
census_T.index.name = 'state'
census_T.reset_index(inplace=True)

# Select only the columns we'll use for analysis
CENSUS_COLS = {
    'Population estimates, July 1, 2016,  (V2016)': 'population_2016',
    'Persons under 18 years, percent, July 1, 2016,  (V2016)': 'pct_under18',
    'Persons 65 years and over, percent,  July 1, 2016,  (V2016)': 'pct_65plus',
    'Female persons, percent,  July 1, 2016,  (V2016)': 'pct_female',
    'White alone, not Hispanic or Latino, percent, July 1, 2016,  (V2016)': 'pct_white',
    'Black or African American alone, percent, July 1, 2016,  (V2016)': 'pct_black',
    'Hispanic or Latino, percent, July 1, 2016,  (V2016)': 'pct_hispanic',
}

census = census_T[['state'] + [c for c in CENSUS_COLS if c in census_T.columns]].copy()
census.rename(columns=CENSUS_COLS, inplace=True)

# Strip commas/percent signs and convert to numeric
def clean_numeric(series):
    """Remove commas/percent signs and cast to float."""
    return pd.to_numeric(series.astype(str).str.replace(',', '', regex=False)
                                           .str.replace('%', '', regex=False)
                                           .str.strip(), errors='coerce')

for col in census.columns[1:]:
    census[col] = clean_numeric(census[col])

print(f"Census shape: {census.shape}")

Merging FBI Gun Data with Census Data¶

The gun dataset has one row per state × month, covering multiple years. Before merging with the census data (which provides a single 2016 population snapshot), we aggregate the gun data by state — summing the total background checks and gun type counts across all months. We then perform an inner join on state name, which drops entries not present in both datasets (e.g., "District of Columbia" and territories absent from the census table).

In [13]:
# Aggregate gun data by state (sum numeric columns across all months)
gun_by_state = (df.groupby('state', as_index=False)
                  .agg(total_checks=('totals', 'sum'),
                       total_handgun=('handgun', 'sum'),
                       total_long_gun=('long_gun', 'sum'),
                       total_permit=('permit', 'sum'),
                       total_multiple=('multiple', 'sum')))

# Inner join — keeps only states present in both datasets
merged = gun_by_state.merge(census, on='state', how='inner')

# Derived column: background checks per 100k residents
merged['checks_per_100k'] = (merged['total_checks'] / merged['population_2016']) * 100_000

Research Question 1: How have FBI gun background checks trended over time, and which states drive the most activity?¶

We'll approach this question from two angles:

  • 1D (time): look at total monthly background checks aggregated nationally to see long-term trends and seasonality.
  • 2D (state × total): identify the top-contributing states and compare them using the merged gun + census data.

Research Question 2: Do state demographics relate to per-capita gun background check rates?¶

Research Question 1 looked at how much and where. Research Question 2 asks why some states are higher than others by comparing the dependent variable checks_per_100k against three demographic independent variables from the census:

  • pct_65plus — percent of population aged 65 and over
  • pct_under18 — percent of population under 18
  • pct_white — percent of non-Hispanic white residents

We'll start with a 1D distribution of per-capita checks, then do 2D scatter plots against each demographic variable, and finish with a correlation matrix to summarize the relationships.

def scatter_vs_checks(x_col, x_label, color):
    """Scatter plot of the given census column against checks_per_100k, with Pearson r."""
    r = merged[x_col].corr(merged['checks_per_100k'])
    plt.figure(figsize=(7, 4))
    plt.scatter(merged[x_col], merged['checks_per_100k'], color=color, alpha=0.7)
    annotate_plot(f'{x_label} vs. Checks per 100k  (r = {r:.2f})',
                  x_label, 'Checks per 100k')

Conclusions¶

Summary of Findings¶

This project investigated the FBI NICS background-check dataset alongside a 2016 U.S. Census snapshot. After cleaning the gun data (parsing month to datetime, filling sparse reporting categories with 0, checking for duplicates) and reshaping the wide-format census table, we aggregated gun totals by state and joined the two sources on state name to produce a 50-state analysis frame.

Research Question 1 — trends over time and top states:

  • National background checks grew substantially between 1998 and 2017, with clear end-of-year seasonal peaks. The single largest month in the data is December 2015 (~3.3M checks).
  • In raw totals, the top states are a mix of large-population states (California, Texas, Florida) and one outlier — Kentucky — which leads absolute totals despite being mid-sized in population.
  • Raw totals correlate moderately with population (Pearson r ≈ 0.65). Once normalized to checks per 100,000 residents, the ranking changes dramatically: Kentucky still leads, followed by Montana, West Virginia, Alaska, and Wyoming.

Research Question 2 — demographics vs. per-capita rate:

  • The per-capita check rate is highly right-skewed across states (skewness ≈ 5.1), driven by a small number of high-outlier states.
  • Among the demographic variables tested, pct_white (non-Hispanic white share) had the strongest positive association with the per-capita rate (r ≈ +0.24). pct_hispanic showed the strongest negative association (r ≈ −0.20). Age-structure variables (pct_65plus, pct_under18) showed only weak associations.

Reflection on the Analysis¶

The wrangling step was where most of the work happened: the two datasets were in very different shapes (long panel vs. wide snapshot), and the gun data had many columns that were sparse by design rather than by accident. Documenting that distinction — and filling those columns with zero instead of dropping rows — preserved the full 50-state sample. Reusing small helper functions (clean_numeric, fill_missing_with_zero, annotate_plot, scatter_vs_checks) kept the analysis cells short and consistent.

Limitations¶

  1. Temporal mismatch between the two datasets. The gun data spans 1998–2017, but the census variables are a single 2016 snapshot. The checks_per_100k metric divides cumulative multi-year totals by one year's population, which overstates the rate for states whose population grew substantially over the window (and understates it for shrinking states). A stricter analysis would restrict gun totals to 2016 only, or interpolate annual population.
  2. Kentucky's policy distortion. Kentucky runs monthly recurring background checks on every active concealed-carry permit holder. This means Kentucky's totals mix "new purchase" checks with routine re-validation checks, which no other state does at this scale. It is the single most influential outlier in every per-capita analysis above.
  3. "Background checks" ≠ "guns purchased." A single transaction can involve multiple firearms, permit transactions include non-purchase activity, and private-party sales are not uniformly captured. The totals column is a count of background-check transactions, not a direct count of guns sold.
  4. Correlational only. All relationships reported above are Pearson correlations on cross-sectional data. No causal claims are made or implied. Unobserved factors — rural/urban composition, gun-ownership culture, state policy regimes — are plausible confounders that were not controlled for.

Suggestions for Further Research¶

  • Restrict the gun data to 2016 only, so that both sides of the ratio cover the same time window, and re-compute every correlation.
  • Flag Kentucky (and any other state with an unusual recurring-check rule) separately, and report findings both with and without it to measure its influence.
  • Bring in additional independent variables — state-level income, urbanization, firearm-law indices — to see whether the weak demographic correlations strengthen or weaken after controlling for those factors.
  • Decompose totals into its sub-components (handgun, long_gun, permit) and ask whether the demographic associations differ by check type.
